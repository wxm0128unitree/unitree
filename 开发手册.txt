机器人设备管理开发文档
1. 核心理念
去流程化：不需要申请、不需要审批。
以状态为中心：所有操作只是为了更新“这台机器现在在哪”和“现在是什么状态”。
一键操作：扫码或点选设备，直接修改状态和去向。
2. 业务逻辑梳理
整个系统只有两个核心动作：
动作一：出库/借出
场景：某人拿走了一台 G1 机器人去测试，或者搬到了二楼实验室。
操作：管理员在列表找到这台机器 -> 点击“借出/出库” -> 输入去向（人名或地点） -> 保存。
结果：机器状态变为“已借出”，状态栏显示“张三”或“二楼实验室”。
动作二：入库/归还
场景：机器送回来了。
操作：管理员找到这台机器 -> 点击“归还/入库”。
结果：机器状态变回“在库”，去向信息清空。
3. 数据库设计
只需要两张表，非常简单。
表一：设备主表
表二：操作日志表 - (可选，用于查账)
4. 界面功能设计
4.1 首页：设备看板
简单的一个列表，一目了然。
筛选栏：
型号下拉框（全部 / G1 / R1 / Go2 / A2）
状态下拉框（全部 / 在库 / 借出 / 维修中）
列表展示：
第一列：资产编号 (建议做成可扫码)
第二列：型号 (标签形式：G1为红色，Go2为蓝色)
第三列：当前状态 (大字显示：🟢在库 / 🔵借出 / 🟠维修)
第四列：去向/持有人 (这是最重要的一栏，如果是维修中，这里写故障原因)
第五列：操作按钮 -> [修改状态]
4.2 核心交互：弹窗修改状态
点击列表中的任意一行设备，或点击“修改状态”，弹出一个极简窗口：
窗口标题：修改状态 - [R-2024-G1-001]
1. 选择状态：
○ 在库 (默认)
○ 借出/使用中
○ 维修中
2. 填写去向： [ ___________________ ]
*(提示：如果选借出，填人名；如果选维修，填故障描述)*
[ 确定 ] [ 取消 ]
5. 操作流程演示
初始：列表显示 G1-001 | 状态：🟢在库 | 去向：-
借出：
扫描 G1-001。
弹窗选择“借出/使用中”。
输入“王五”。
点击确定。
列表更新：G1-001 | 状态：🔵借出 | 去向：王五
转移：
王五把机器给到了赵六。
管理员点击 G1-001，状态保持“借出”，修改去向为“赵六”。
维修：
机器坏了。
管理员点击 G1-001，选择“维修中”。
输入“左腿电机故障”。
列表更新：G1-001 | 状态：🟠维修 | 去向：左腿电机故障
6. 给开发者的技术提示
开发框架：建议用最简单的 Web 前端 (Vue/React) + Python (Flask/FastAPI) 或者 PHP。
无需登录（如果在内网安全环境）：直接打开网页就能操作，提高速度。
移动端适配：界面一定要做成手机能用的（响应式），因为管理员拿着手机在仓库里走动操作的场景最多。
